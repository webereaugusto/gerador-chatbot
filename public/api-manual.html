Manual da API externa (v1)

Integração somente leitura para sistemas externos (n8n, CRM, scripts). Todas as rotas abaixo exigem uma chave de API gerada no painel em API (formato gc_live_…).

1. Base URL e autenticação

Substitua SEU_DOMINIO pela URL do seu deploy (ex. gerador-chatbot-alpha.vercel.app). Use sempre HTTPS em produção.

Envie a chave em um destes formatos:

Opção Header
recomendado Authorization: Bearer gc_live_…
alternativa X-Api-Key: gc_live_…

401 — chave ausente, inválida ou revogada.
503 — servidor sem API_KEY_PEPPER configurado (veja README / painel).

2. Endpoints

GET /api/v1/chatbots

Lista os chatbots da conta vinculada à chave. Não expõe chaves OpenAI, Evolution nem base de conhecimento.

Resposta 200 (exemplo):

{
  "items": [
    {
      "id": "uuid",
      "name": "Franquias",
      "createdAt": "2026-04-18T…",
      "configured": true,
      "webhookUrl": "https://SEU_DOMINIO/webhook/evolution/uuid"
    }
  ]
}

O campo webhookUrl usa o host da requisição. Em integrações server-side, o host costuma ser o do seu projeto na Vercel.

GET /api/v1/chatbots/:chatbotId/leads

Lista leads de um chatbot. Só retorna dados se o chatbot pertencer à mesma conta da chave.

Query Descrição
limit 1 a 100 (padrão 50)
before ISO 8601 — última mensagem do lead anterior à data (paginação por last_message_at). Use o valor nextBefore da resposta anterior quando hasMore for true.

Resposta 200:

{
  "items": [ { "id", "chatbot_id", "phone", "name", "source", "last_message_at", "created_at" } ],
  "nextBefore": "2026-04-18T12:00:00.000Z",
  "hasMore": false
}

404 — chatbot inexistente ou de outro usuário.

GET /api/v1/leads/:leadId/messages

Retorna o lead (metadados) e todas as mensagens em ordem cronológica (usuário + assistente).

Resposta 200:

{
  "lead": {
    "id", "chatbot_id", "phone", "name", "source",
    "last_message_at", "created_at"
  },
  "items": [
    { "id", "lead_id", "role": "user" | "assistant", "content", "created_at" }
  ]
}

404 — lead inexistente ou não pertence à sua conta.

3. Exemplos com curl

Listar chatbots:

curl -sS -H "Authorization: Bearer gc_live_SUA_CHAVE" \
  "https://SEU_DOMINIO/api/v1/chatbots"

Listar leads (com limite):

curl -sS -H "X-Api-Key: gc_live_SUA_CHAVE" \
  "https://SEU_DOMINIO/api/v1/chatbots/UUID_DO_CHATBOT/leads?limit=20"

Próxima página de leads:

curl -sS -H "Authorization: Bearer gc_live_SUA_CHAVE" \
  "https://SEU_DOMINIO/api/v1/chatbots/UUID_DO_CHATBOT/leads?limit=20&before=VALOR_NEXT_BEFORE"

Mensagens de um lead:

curl -sS -H "Authorization: Bearer gc_live_SUA_CHAVE" \
  "https://SEU_DOMINIO/api/v1/leads/UUID_DO_LEAD/messages"

4. CORS

As rotas /api/v1/* enviam Access-Control-Allow-Origin: *, permitindo chamadas a partir de outras origens (útil para testes no navegador). Em produção, prefira chamar a API a partir do seu backend para não expor a chave no front-end público.

5. Segurança

  • Trate a chave como senha: não commite em repositórios nem em prints.
  • Revogue no painel se vazar.
  • A API v1 é apenas GET — não altera dados.
